City Road Network Simulation

Write a simple Python script that loads the road network for Berlin from the OpenStreetMaps API into a NetworkX graph.

To load the road network for Berlin from the OpenStreetMaps (OSM) API into a NetworkX graph, we can utilize the osmnx library, which simplifies this process. Here's a simple script to achieve this:

  1. First, you'll need to install the necessary libraries:
bash
pip install osmnx
  1. Next, you can use the following Python script:
python
import osmnx as ox
import networkx as nx
import matplotlib.pyplot as plt

def load_berlin_road_network():
    # Get the road network for Berlin
    G = ox.graph_from_place('Berlin, Germany', network_type='drive')
    
    return G

def plot_network(G):
    fig, ax = ox.plot_graph(G, figsize=(10,10))
    
    return fig, ax

if __name__ == "__main__":
    berlin_network = load_berlin_road_network()
    plot_network(berlin_network)

Running the script will fetch the road network for Berlin and plot it. The network_type='drive' argument ensures you're getting only the drivable roads. You can modify this to other types like walk, bike, etc., if you're interested in other kinds of road networks.

Remember, the OSM API and osmnx library provide a lot of flexibility. You can fetch road networks for other cities, specify types of roads, fetch building footprints, and more.

Create a simple traffic simulation that can run on a directed graph like the one above.

Certainly! A basic traffic simulation can be designed based on the following principles:

  1. Cars move along edges of the graph from one node to the next.
  2. Each car has a starting node and a destination node.
  3. Cars choose the shortest path (or another criterion) from start to destination.
  4. Edges have capacities: if too many cars use the same edge, a "traffic jam" occurs, slowing down the cars.

Here's a simplified Python script to demonstrate this:

python
import osmnx as ox
import networkx as nx
import random

class Car:
    def __init__(self, start, destination):
        self.current_location = start
        self.destination = destination
        self.path = []

def move_cars(G, cars):
    for car in cars:
        if not car.path:
            # Determine path based on shortest route
            car.path = nx.shortest_path(G, car.current_location, car.destination, weight="travel_time")
        
        # Check if car has reached its destination
        if car.current_location == car.destination:
            continue

        next_node = car.path.pop(0)
        
        # Simulate traffic jam: if there are more than 'x' cars on an edge, slow down
        cars_on_edge = sum(1 for c in cars if c.current_location == car.current_location and c.path and c.path[0] == next_node)
        if cars_on_edge > 5:  # Arbitrary threshold for a "jam"
            car.path.insert(0, next_node)  # Car stays in the same spot
        else:
            car.current_location = next_node

def simulate_traffic(G, num_cars=100, num_steps=10):
    # Create cars with random starts and destinations
    nodes = list(G.nodes())
    cars = [Car(random.choice(nodes), random.choice(nodes)) for _ in range(num_cars)]
    
    for step in range(num_steps):
        move_cars(G, cars)

        # For demonstration, print the number of cars at each node
        node_counts = {node: 0 for node in nodes}
        for car in cars:
            node_counts[car.current_location] += 1
        print(f"Step {step + 1}:")
        for node, count in node_counts.items():
            if count:
                print(f"Node {node}: {count} cars")

if __name__ == "__main__":
    G = ox.graph_from_place('Berlin, Germany', network_type='drive')
    
    # Add a 'travel_time' attribute to each edge to use as weight (for now, assume time is inversely proportional to speed limit, with a default of 30 km/h)
    for u, v, k, data in G.edges(data=True, keys=True):
        speed = data.get("maxspeed", 30)
        if isinstance(speed, list):  # There can be multiple speed limits; take the first one as representative
            speed = speed[0]
        data["travel_time"] = data["length"] / (float(speed) * 1000/60)  # length in meters, speed in km/h, result in minutes

    simulate_traffic(G)

This script represents a very rudimentary traffic simulation and makes numerous simplifications. In practice, traffic simulations consider many more factors, such as varying car speeds, traffic lights, different types of vehicles, and intricate driver behaviors. Tools like SUMO (Simulation of Urban MObility) offer more realistic simulations. If you're aiming for a detailed simulation, it's worth looking into such specialized tools.

Write a function to visualize the state of the simulation. Show the number of cars on each edge of the graph.

To visualize the state of the simulation, we can modify the edge colors and widths based on the number of cars on each edge. Thicker edges and stronger colors can represent higher car densities.

Here's how you can do this:

  1. Add a function to calculate the number of cars on each edge.
  2. Modify the plot_network function to visualize the edge car counts.

import networkx as nx

import matplotlib.pyplot as plt

    # ... [rest of the Car class definition remains the same]

def edge_car_counts(G, cars):
    counts = {(u, v, 0): 0 for u, v, _ in G.edges(keys=True)}
    for car in cars:
        if car.path:
            next_node = car.path[0]
            counts[(car.current_location, next_node, 0)] += 1
    return counts

def plot_network_with_traffic(G, cars):
    edge_colors = []
    edge_widths = []

    counts = edge_car_counts(G, cars)
    max_count = max(counts.values())

    for _, _, _, data in G.edges(data=True, keys=True):
        count = counts.get((data['osmid'], data['osmid'], 0), 0)
        edge_colors.append(plt.cm.Reds(count / max_count))
        edge_widths.append(1 + count)

    fig, ax = ox.plot_graph(G, figsize=(10,10), edge_color=edge_colors, edge_linewidth=edge_widths)
    return fig, ax

The plot_network_with_traffic function calculates the number of cars on each edge and then sets edge colors and widths based on those counts. In this case, I've used the Reds colormap to represent car densities, but you can choose any other colormap as well.
